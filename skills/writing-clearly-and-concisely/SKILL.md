# 清晰简洁地写作

## 概览

本 skill 用于把重要正文改得清楚、简短、具体。核心原则：先让读者准确理解，再删掉不必要的词。

它只处理表达质量；中文技术文档排版、术语保留、链接格式和中英混排细节，遵守 chinese-documentation-guide.md。

## 何时使用

- 用户明确要求润色、压缩、改写、整理或提高表达质量。
- 发布前打磨重要文档、README、技术解释、报告或摘要。
- 提交前打磨 commit message、pull request description、release note。
- 发布前打磨 error message、UI copy、help text、重要 comment。
- 当前文字质量会影响读者理解、决策或执行。

不要因为普通回复、进度更新或短结论自动使用本 skill；这类文字直接写清楚即可。

## 快速参考

| 情况               | 处理                                |
| ------------------ | ----------------------------------- |
| 冗长               | 删除铺垫、套话、重复限定            |
| 含糊               | 换成具体对象、动作、结果            |
| 难扫读             | 拆短句，按步骤或要点重排            |
| 语气虚             | 用主动语态和肯定表达                |
| 中文排版或术语问题 | 参考 chinese-documentation-guide.md |

## 核心模式

1. 明确读者和发布位置。
2. 保留事实、约束、结论和必要语气。
3. 删除不改变含义的词句。
4. 把抽象判断改成可执行或可验证的表达。
5. 把强调内容放在读者最容易看到的位置。

## 常见错误

- 只缩短字数，却删掉了范围、风险或条件。
- 把原文改得更正式，但没有更清楚。
- 为了"简洁"省略动作主体，导致责任不明。
- 对中文技术文档问题重复解释规则，而不是遵守 chinese-documentation-guide.md。

## 上下文紧张时

长文本或发布质量要求高、且当前上下文不适合直接处理时，才派 subagent。提供 draft、目标读者和发布位置，要求它返回修订版和关键修改理由；涉及中文技术文档规范时，要求它遵守 chinese-documentation-guide.md。

## 底线

不要把润色变成改写事实。重要正文先准确，再简洁；普通短回复不需要启动重流程。
