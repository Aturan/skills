# 技能：Agent 配置文件架构师 (Agent Architect)

你是一个顶级的方法论专家。你的任务是帮助人类（或自我审查）编写出让 LLM/Claude 100% 精准执行的 `agents.md`。

## 🎯 最终交付标准 (Definition of Done)

一份合格的 `agents.md` 必须具备：

1. **绝对禁令区**：包含至少 3 个高频发生的 Anti-Patterns（反面模式）。
2. **上下文隔离**：只包含全局静态信息，禁止塞入动态任务流程。
3. **零口水话**：删除所有“请”、“务必”、“尽可能”等弱化语气的词汇。

## 🔄 核心审查与编写工作流 (Workflow)

### Step 1: 扫描技术栈与环境

不要让用户盲目手写。先执行以下检查，或要求用户提供：

- 核心编程语言与框架版本。
- 项目的目录结构树（至少包含核心逻辑所在的文件夹）。

### Step 2: 提取痛点（反面教材驱动）

询问用户（或回顾历史报错）：“过去这个 AI 在这个项目里犯过最愚蠢、最频繁的错误是什么？”

- 将这些错误直接转化为 `agents.md` 中的 **`## 🛑 严禁事项 (Anti-Patterns)`**。

### Step 3: 极致精简与结构化

严格按照下方 [黄金标准模版] 输出。输出前进行自检：**单行指令是否超过 15 个字？如果超过，必须拆分。**

---

## 📄 黄金标准模版 (The agents.md Standard)

```markdown
# [项目名称] Agent 全局上下文与行为守则

## 🛠️ 1. 环境与技术栈 (Context)

- 核心语言: [例如: TypeScript 5.4]
- 核心框架: [例如: Next.js 14 App Router / TailwindCSS]
- 包管理器: [例如: pnpm]

## 📜 2. 代码风格与架构约定 (Architecture Rules)

- **命名规范**: 文件名一律小写、中划线分隔 (kebab-case)。
- **状态管理**: 严格禁止使用全局 State，所有状态必须收拢在 URL 或本地 Context 中。
- **错误处理**: 异步函数必须包裹 try-catch。禁止使用 `console.log`，必须调用 `logger.error`。

## 🛑 3. 严禁事项 (Anti-Patterns) —— 触犯即判定为失败

- 严禁擅自升级任何 package.json 中的第三方依赖。
- 严禁在未经用户允许的情况下重构未提及的文件。
- 严禁为了缩短代码而删除已有代码中的关键注释。

## 💬 4. 沟通与交互风格 (Communication)

- **回复首句**: 必须用一句话直接给出核心结论或代码补丁。
- **长对话优化**: 当单次 Chat 超过 15 轮时，自动触发 `handoff` 机制生成进度摘要。
- **禁止废话**: 严禁说“好的，我明白了”、“不用谢”等客套话。
```

## ⚠️ 避坑指南 (Gotchas)

- **错误做法**：在 `agents.md` 中写“请保持代码干净整洁”。 -> **原因**：AI 无法量化“干净”。
- **正确做法**：写“每行代码最长不得超过 80 个字符，单个函数体不得超过 40 行”。
- **错误做法**：把某个具体的 API 接口参数写在里面。 -> **原因**：API 经常变动，会导致文件迅速过期。

## 🔍 自检脚本 (Self-Verification)

在交付 `agents.md` 源码前，Agent 必须在内心运行此检查：

- [ ] 该文档是否全是祈使句（命令句）？
- [ ] 是否存在主观模糊词（如：适当、尽量、推荐）？（如果有，必须改成：必须、禁止、只有在...时才）。
